Написать REST-сервис истории переводов

Структура БД:
База данных содержит 2 таблицы:
1. Таблица лимитов содержит следующие колонки
· Id
· Страна назначения перевода
· Валюта перевода
· Максимальный объем перевода в месяц
2. Таблица истории переводов содержит следующие колонки
· Id
· Дата перевода
· Сумма перевода
· Валюта перевода
· Страна назначения перевода

API сервиса
Сервис должен предоставлять полный набор CRUD-запросов (GET, POST, PUT, DELETE) для работы с таблицей лимитов, а также
запрос на внесение операции в историю, если лимит для данного клиента не был/не будет превышен в текущем месяце.
Валюта – строковое сокращение. Допустимые значения:
Рубли       RUB
Доллары     USD
Евро        EUR
Страна – строковое сокращение. Допустимые значения:
Россия      RUS
Абхазия     ABH
Австралия   AUS
Пример заполнения таблицы:
CLIENT_ID   DATE                    AMOUNT  CUR     COUNTRY
258963      2018-12-03 16:58:02     15000   USD     AUS
258963      2018-12-31 16:58:02     50000   USD     ABH
456123      2019-02-03 00:26:45     100000  RUB     RUS

Требования к реализации:
Суммы переводов хранятся в «копейках», следует реализовать тесты с помощью pytest на все методы API. Технологии:
aiohttp, postgresql, alembic, pytest
· Сервис следует реализовать на aiohttp.
· В качестве базы данных нужно использовать Postgres.
· Для работы с базой должен быть реализован скрипт, инициализирующий ее структуру.
· По каждому из запросов необходимо описать формат запроса и ответа на запрос в любом удобном для вас виде (А если с
помощью формата openapi, то будет плюсом)
· Код следует разместить на github.
